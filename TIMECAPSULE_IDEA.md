# 커플 타임캡슐 기능 아이디어 (진행 중 — 아직 미구현)

여행 중 매일 각자 질문에 답을 남기고, 서로 모르게 쌓아뒀다가 여행이 끝난 뒤 함께 열어보는 기능. 다음 세션에서 이어서 디벨롭할 것.

## 확정된 설계

- **기기 구성**: 한 폰을 번갈아 쓰는 게 아니라, **둘 다 자기 휴대폰에 각자 앱을 설치**해서 각자 입력.
  - "몰래 입력" UI가 따로 필요 없음 (구조가 단순해짐)
  - 기기간 동기화가 없어도 되므로 Zero-Data 원칙과 자연스럽게 맞음
  - 오픈 시점에 실제로 "서로의 폰을 처음 보여주는" 이벤트가 됨
  - 단점: 둘 다 앱을 각자 설치해야 하고, 오늘 일정 등 공용 정보도 각자 폰에서 따로 봐야 함(다만 현재 앱도 이미 그럴 가능성이 높은 구조)

- **답변 잠금 규칙**: 그날 자정까지는 수정 가능하고, 다음날로 넘어가면 그날 답은 잠김 — 나중에 고쳐 쓰지 않고 "그 순간의 진짜 감정"이 그대로 남도록 함.

- **질문 방식**: 매일 새로운 40개(10문항×4일)를 짜기보다, **고정된 질문 10개를 매일 반복**해서 묻는 방식으로 결정 (콘텐츠 제작 부담이 적고, 매일 같은 질문에 답하는 것 자체가 자연스러운 리추얼이 됨).

## 질문 10개 (초안 — 아직 사용자 확정 대기)

사용자가 예시로 준 4개(1~4번)에 6개를 더 제안한 상태. 다음 세션에서 최종 확정 필요.

1. 오늘 가장 맛있었던 음식은?
2. 오늘 제일 행복했던 순간은?
3. 오늘 제일 미안했던 순간은?
4. 오늘을 마무리하며 가장 해주고 싶은 말은?
5. 오늘 상대방이 제일 예뻐/멋있어 보였던 순간은?
6. 오늘 가장 웃겼던 일은?
7. 오늘 하루를 색깔로 표현한다면?
8. 오늘 제일 고마웠던 순간은?
9. 내일은 뭘 하고 싶어?
10. 오늘 우리 사이에 새롭게 알게 된 점은?

가벼운 질문(맛있었던 음식, 웃겼던 일)과 깊은 질문(미안했던 순간, 새롭게 알게 된 점)을 섞어 구성함. 사용자 피드백 대기 중 — 빼거나 바꾸고 싶은 문항 있을 수 있음.

## 아직 결정 안 된 것 (다음 세션에서 확인)

- **오픈(공개) 시점**: "귀국 다음날 아침"을 제안했으나 사용자 확답 대기 중. 다른 후보: 상해 마지막 밤(7/19), 귀국 당일 저녁 등.
- **오픈 연출**: 봉인된 타임캡슐(항아리 또는 편지봉투 모양, 망고 캐릭터와 같은 톤의 SVG)이 오픈 시점 전엔 잠겨있다가 날짜가 지나면 자동으로 열리는 화면을 제안했음 — 구체적 비주얼/인터랙션은 미정.
- 질문 10개 최종 컨펌.

## 구현 시 참고할 기존 패턴 (아직 코드 작업 전, 참고용 메모)

- localStorage 저장/로드 패턴은 `src/components/CurrencyConverter.jsx`의 `loadRate()` + `useEffect` 저장 구조를 그대로 따르면 됨.
- 오픈 여부 판단은 오늘 날짜와 `travelData.meta.endDate`(또는 별도 오픈 기준일)를 비교하는 방식.
- 새 데이터는 스팟과 무관하므로 `travelData.js`에 별도 최상위 키(예: `timeCapsule.questions`)로 질문 목록만 정적으로 두고, 답변 자체는 전부 localStorage에 사용자별로 저장.
- 디자인은 기존 `@theme` 파스텔 팔레트 그대로 사용, 새 SVG는 `MangoCouple.jsx`/`WeatherForecast.jsx`처럼 인라인 SVG + CSS keyframes로 구현 (Zero-Data 원칙 유지).
